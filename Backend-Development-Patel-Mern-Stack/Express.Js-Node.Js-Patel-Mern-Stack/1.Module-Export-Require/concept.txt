In Node.js, a module is a reusable piece of code that can be exported from one file and imported into another. 
Modules allow developers to organize their code into smaller, manageable, and reusable components, making applications easier to 
maintain and scale.


Why Use Modules?
Code Organization:
Helps structure the codebase logically.
Reusability:
Write once and reuse in multiple parts of the application.
Encapsulation:
Avoids polluting the global namespace by scoping variables and functions.
Maintainability:
Smaller, modular code is easier to debug and update.



What is CommonJS?
CommonJS is a module system used in JavaScript, especially in Node.js, for organizing code into smaller, reusable pieces (called modules). 
It allows developers to use require() to import functionality and module.exports to export it.

Think of it as a way to break your code into "small boxes" (modules) and then use those boxes wherever needed.

Why Was CommonJS Created?
In early JavaScript development, there was no standardized way to include reusable code from other files. Developers used global variables or manually copy-pasted code, which caused issues like:

Name collisions: Multiple scripts using the same variable names.
Unorganized code: Difficult to manage large projects.
CommonJS solved this problem by introducing:

A clear way to import modules (require).
A clear way to export modules (module.exports).

Features of CommonJS
Synchronous Loading:

Modules are loaded and executed synchronously.
This means the script stops and waits for the module to load before continuing execution.
Single Exports Object:

Each module in CommonJS exports a single object (module.exports), which can contain multiple values.
Module Caching:

Once a module is loaded, Node.js caches it to improve performance. If you require() the same module again, it won’t reload—it uses the cached version.
File-Based Modules:

Each file is treated as a separate module.
